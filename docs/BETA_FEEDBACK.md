# Beta Feedback Collection

Thank you for testing Telos Idea Matrix! Your feedback is crucial for making v1.0 the best it can be.

## How to Provide Feedback

### 1. Bug Reports
Use GitHub Issues: https://github.com/rayyacub/telos-idea-matrix/issues/new

**Template**:
```markdown
**Bug Title**: [Brief description]

**Environment**:
- OS: [Linux/macOS/Windows]
- Docker Version: [run: docker --version]
- TIM Version: [run: docker exec telos-api-staging tm --version]

**Steps to Reproduce**:
1. [First step]
2. [Second step]
3. [etc.]

**Expected Behavior**: [What should happen]
**Actual Behavior**: [What actually happened]

**Logs** (if applicable):
```
[Paste relevant logs here]
```

**Screenshots** (if applicable): [Attach images]

**Priority**: [Low/Medium/High/Critical]
```

### 2. Feature Requests
Use GitHub Issues with `[FEATURE]` tag

**Template**:
```markdown
**Feature Request**: [Feature name]

**Problem Statement**:
[What problem does this solve?]

**Proposed Solution**:
[How should it work?]

**Alternatives Considered**:
[Other ways to solve this]

**Use Case**:
[When would you use this feature?]

**Priority**: [Nice to have / Important / Critical for v1.0]
```

### 3. Daily Check-ins
Post brief updates in GitHub Discussions: https://github.com/rayyacub/telos-idea-matrix/discussions

**Format**:
```markdown
## Day [X] Check-in - [Your Name]

**What I tested today**:
- [Feature 1]
- [Feature 2]

**What worked well**:
- ✅ [Something good]

**What didn't work**:
- ❌ [Issue 1] - [Filed as Issue #XX]
- ❌ [Issue 2] - [Filed as Issue #YY]

**Questions**:
- [Any questions for the team?]

**Tomorrow's focus**:
- [What you'll test next]
```

### 4. Final Survey
Complete at the end of beta period (Day 7)

## Feedback Analysis

We'll aggregate and analyze all feedback to:

1. **Prioritize bugs** for fixing before v1.0
2. **Identify patterns** in user experience
3. **Determine feature priorities** for v1.0 and beyond
4. **Measure success criteria**:
   - >80% setup success rate
   - >80% satisfaction (scores of 4-5)
   - <5 critical bugs
   - >90% would continue using

Results will be shared with all beta testers before v1.0 launch!
